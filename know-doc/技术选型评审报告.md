## 二、版本升级方案

### 2.1 SpringBoot 2.6 → 3.2 升级检查清单

```
✅ 1. Java 版本必须 17+
✅ 2. javax.* → jakarta.* 命名空间迁移
✅ 3. Spring Security 配置变化（SecurityFilterChain）
✅ 4. 第三方库兼容性检查
✅ 5. 配置文件格式变化（可选）
✅ 6. Actuator 端点变化
```

## 三、组件精简建议

### 3.1 链路追踪选型

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| Skywalking | 功能全、UI好 | 资源消耗大 | 生产环境 |
| Zipkin | 轻量、易集成 | 功能简单 | 微小系统 |
| Pinpoint | 无侵入 | 仅支持Java |  APM场景 |

**推荐：Skywalking**

### 3.2 服务限流选型

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| Sentinel | 阿里维护、功能全 | 学习曲线 | 复杂限流 |
| Hystrix | Spring原生 | 停止维护 | 简单场景 |

**推荐：Sentinel（统一限流 + 熔断 + 降级）**

### 3.3 最终组件清单

```
微服务基础设施
├── nacos              -- 注册中心+配置中心（推荐2023版本）
├── gateway            -- 网关
├── Sentinel           -- 限流+熔断+降级（统一）
├── Seata              -- 分布式事务（保留）
├── jetcache           -- 二级缓存（保留）
└── Skywalking         -- 链路追踪（统一）

中间件
├── MySQL 8.0          -- 主数据库
├── Redis              -- 缓存+分布式锁
├── RabbitMQ           -- 消息队列（保留）
├── Elasticsearch      -- 全文检索（保留）
└── Flyway             -- 版本管理（保留）
```

## 四、风险评估

### 4.1 升级风险

| 风险项 | 影响程度 | 缓解措施 |
