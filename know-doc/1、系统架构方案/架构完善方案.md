## 一、API 规范设计

### 1.1 请求格式

```json
{
  "code": 200,
  "message": "success",
  "data": {
    // 业务数据
  },
  "timestamp": 1704067200000,
  "requestId": "uuid-xxxx"
}
```

### 1.2 统一响应包装

```java
@Data
@ApiModel("统一响应")
public class ResultMessage<T> implements Serializable {
    
    @ApiModelProperty("状态码")
    private Integer code;
    
    @ApiModelProperty("提示信息")
    private String message;
    
    @ApiModelProperty("业务数据")
    private T data;
    
    @ApiModelProperty("时间戳")
    private Long timestamp;
    
    @ApiModelProperty("请求ID")
    private String requestId;
    
    public static <T> ResultMessage<T> success(T data) {
        ResultMessage<T> result = new ResultMessage<>();
        result.setCode(200);
        result.setMessage("success");
        result.setData(data);
        result.setTimestamp(System.currentTimeMillis());
        return result;
    }
    
    public static <T> ResultMessage<T> error(String message) {
        return error(500, message);
    }
    
    public static <T> ResultMessage<T> error(Integer code, String message) {
        ResultMessage<T> result = new ResultMessage<>();
        result.setCode(code);
        result.setMessage(message);
        result.setTimestamp(System.currentTimeMillis());
        return result;
    }
}
```

## 二、数据库设计规范

### 2.1 命名规范

| 类型 | 规则 | 示例 |
|------|------|------|
| 表名 | 小写+下划线 | sys_user |
| 字段名 | 小写+下划线 | user_name |
| 主键 | id 或 表名_id | id, user_id |
| 外键 | 关联表_id | dept_id |
| 索引 | idx_字段名 | idx_user_name |
| 唯一索引 | uk_字段名 | uk_user_name |

### 2.2 基础表结构

```sql
-- 所有业务表的公共字段
CREATE TABLE `template` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键',
  `tenant_id` bigint DEFAULT NULL COMMENT '租户ID',
  `create_by` bigint DEFAULT NULL COMMENT '创建人',
  `create_time` bigint DEFAULT NULL COMMENT '创建时间',
  `update_by` bigint DEFAULT NULL COMMENT '更新人',
  `update_time` bigint DEFAULT NULL COMMENT '更新时间',
  `delete_time` bigint DEFAULT NULL COMMENT '删除时间',
  `remark` varchar(500) DEFAULT NULL COMMENT '备注',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='业务表模板';
```

### 2.3 连接池配置

```yaml
spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    druid:
      # 初始连接数
      initial-size: 5
      # 最大连接数
      max-active: 20
      # 最小空闲连接
      min-idle: 5
      # 获取连接等待超时时间
      max-wait: 60000
      # 检测间隔
      time-between-eviction-runs-millis: 60000
      # 最小生存时间
      min-evictable-idle-time-millis: 300000
      # 检测有效连接
      validation-query: SELECT 1
      # 取出空闲连接
      remove-abandoned: true
      # 超时时间
      remove-abandoned-timeout: 1800
```

## 三、微服务设计

### 3.1 服务划分

```
├── gateway              -- 网关服务（端口: 8080）
├── system               -- 系统服务（端口: 8081）
│   ├── user            -- 用户管理
│   ├── auth            -- 认证授权
│   ├── dept            -- 部门管理
│   ├── role            -- 角色管理
│   └── menu            -- 菜单管理
├── business             -- 业务服务（端口: 8082）
├── file                 -- 文件服务（端口: 8083）
└── monitor              -- 监控服务（端口: 8084）
```

### 3.2 网关路由配置

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: system-service
          uri: lb://system
          predicates:
            - Path=/system/**
          filters:
            - StripPrefix=1
            
        - id: business-service
          uri: lb://business
          predicates:
            - Path=/business/**
          filters:
            - StripPrefix=1
```

### 3.3 服务鉴权

## 四、安全设计

### 4.1 HTTPS 配置

```yaml
server:
  ssl:
    enabled: true
    key-store: classpath:ssl.p12
    key-store-password: password
    key-store-type: PKCS12
```

### 4.2 敏感字段加密

```java
@Data
public class User {
    
    @JsonIgnore
    private String password;
    
    @Sensitive(field = Sensitive.Field.ENCRYPT)
    private String phone;
    
    @Sensitive(field = Sensitive.Field.MASK)
    private String idCard;
}
```

### 4.3 接口签名验签

```
请求参数：
- appId: 应用ID
- timestamp: 时间戳
- nonce: 随机数
- sign: 签名（MD5(appId+timestamp+nonce+secret)）

签名算法：
sign = MD5(appId + timestamp + nonce + secret)
```

## 五、日志规范

### 5.1 日志级别

| 级别 | 使用场景 |
|------|----------|
| ERROR | 程序异常，需要人工处理 |
| WARN | 警告，不影响功能但需关注 |
| INFO | 正常业务流程记录 |
| DEBUG | 调试信息，生产环境关闭 |

### 5.2 日志格式

```
[%d{yyyy-MM-dd HH:mm:ss.SSS}] 
[%thread] 
[%-5level] 
[%logger{36}] 
[%X{traceId}] 
- %msg%n
```

### 5.3 TraceId 传递

```java
// MDC 设置
MDC.put("traceId", traceId);

// Feign 传递
@PostMapping(value = "/xxx")
ResultMessage<Void> xxx(@RequestHeader("X-Trace-Id") String traceId);
```
