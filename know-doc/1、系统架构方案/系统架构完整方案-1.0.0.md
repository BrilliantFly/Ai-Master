## 一、项目概述

### 1.1 项目定位

企业级全栈后台管理系统，包含 Web 管理端、移动端（微信小程序）、Java 微服务后端 + Python 服务。

## 三、API 规范设计

### 3.1 统一请求格式

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1704067200000,
  "requestId": "uuid-xxxx"
}
```

### 3.2 分页响应

```json
{
  "code": 200,
  "data": {
    "list": [],
    "total": 100,
    "pageNum": 1,
    "pageSize": 10
  }
}
```

### 3.3 错误码规范

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 3.4 接口签名

```
请求参数：
- appId: 应用ID
- timestamp: 时间戳
- nonce: 随机数
- sign: 签名

签名算法：sign = MD5(appId + timestamp + nonce + secret)
```

## 五、权限管理设计

### 5.1 核心表结构

| 表名 | 说明 |
|------|------|
| sys_user | 用户表 |
| sys_role | 角色表 |
| sys_permission | 权限表 |
| sys_menu | 菜单表 |
| sys_user_role | 用户-角色关联表 |
| sys_role_permission | 角色-权限关联表 |
| sys_role_menu | 角色-菜单关联表 |
| sys_menu_permission | 菜单-权限关联表 |
| sys_dept | 部门表 |
| sys_post | 岗位表 |
| sys_tenant | 租户表 |
| sys_dict_type | 字典类型表 |
| sys_dict_data | 字典数据表 |

### 5.2 权限类型

| 权限类型 | 说明 | 实现方式 |
|----------|------|----------|
| 菜单权限 | 控制菜单访问 | sys_menu + sys_role_menu |
| 按钮权限 | 控制操作权限 | sys_permission + 按钮标识 |
| 数据权限 | 行级数据控制 | sys_data_permission |
| 租户权限 | 多租户隔离 | tenant_id 字段 |

### 5.3 认证流程

```
用户登录 → 验证用户名密码 → 生成JWT Token → 返回前端
        ↓
前端请求携带Token → 网关验证 → 服务校验权限 → 响应
```

### 5.4 Token 配置

| 配置项 | 推荐值 |
|--------|--------|
| Access Token 有效期 | 2小时 |
| Refresh Token 有效期 | 7天 |
| 签名算法 | HS256 |
| 存储方式 | httpOnly Cookie |

## 六、数据字典设计

### 6.1 表结构

```sql
-- 字典类型表
CREATE TABLE `sys_dict_type` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `dict_name` varchar(100) NOT NULL COMMENT '字典名称',
  `dict_type` varchar(100) NOT NULL COMMENT '字典类型',
  `remark` varchar(500) DEFAULT NULL,
  `status` tinyint DEFAULT 1 COMMENT '状态',
  `create_time` bigint,
  `update_time` bigint,
  `delete_time` bigint,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_dict_type` (`dict_type`)
);

-- 字典数据表
CREATE TABLE `sys_dict_data` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `dict_type_id` bigint NOT NULL,
  `dict_label` varchar(100) NOT NULL,
  `dict_value` varchar(100) NOT NULL,
  `dict_code` varchar(100) DEFAULT NULL,
  `sort` int DEFAULT 0,
  `status` tinyint DEFAULT 1,
  `remark` varchar(500) DEFAULT NULL,
  `create_time` bigint,
  `update_time` bigint,
  `delete_time` bigint,
  PRIMARY KEY (`id`),
  KEY `idx_dict_type_id` (`dict_type_id`)
);
```

### 6.2 缓存策略

| 缓存Key | 过期时间 | 失效时机 |
|---------|----------|----------|
| dict:{dictTypeId} | 30分钟 | 增删改 |
| dict:{dictTypeCode} | 30分钟 | 增删改 |

## 七、安全设计

### 7.1 接口签名验签

```
签名算法：sign = MD5(appId + timestamp + nonce + secret)
```

### 7.2 敏感字段加密

```java
@Sensitive(field = Sensitive.Field.ENCRYPT)
private String phone;

@Sensitive(field = Sensitive.Field.MASK)
private String idCard;
```

### 7.3 HTTPS 配置

```yaml
server:
  ssl:
    enabled: true
    key-store: classpath:ssl.p12
    key-store-password: password
    key-store-type: PKCS12
```

### 7.4 密码安全

| 配置项 | 推荐值 |
|--------|--------|
| 加密算法 | BCrypt |
| 密码长度 | 最少8位 |
| 错误锁定 | 连续5次错误锁定15分钟 |
| 密码过期 | 90天 |
