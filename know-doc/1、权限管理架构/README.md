权限管理架构
===============

当前最新版本： 0.0.1

### 核心功能模块
* 核心模块
  - 租户管理
  - 用户管理
  - 角色管理
  - 菜单/资源管理
  - 权限/资源管理
* 日志模块
  - 登录日志
  - 操作日志
* 其他模块
  - 部门/组织管理
  - 岗位管理
  - 数据字典管理

### 关键实现细节
* 数据库设计其中的5张表：用户表、角色表、权限表、用户-角色关联表、角色-权限关联表。通过这种多对多的关系，可以实现灵活的权限分配。
* 认证流程（JWT）： 用户通过登录接口提交用户名和密码。 后端Spring Security验证成功后，生成一个包含用户唯一标识和权限信息的JWT字
  符串返回给前端。 前端在后续请求的Authorization头中携带该Token。 后端编写一个JWT过滤器，拦截请求并解析Token，如果解析成功则将
  用户信息存入SecurityContext，代表认证通过。
* 权限
  - 动态权限控制： 权限控制不应是硬编码在代码里的（如@PreAuthorize("hasRole('ADMIN')")），更灵活的方式是将“访问路径”与“所需权
    限”的对应关系存储在数据库中。当请求到达时，系统动态地从数据库或缓存（Redis）中查询当前请求的URL需要什么权限，再与当前JWT Token
    中用户的权限进行比对，从而决定是否放行
  - 权限颗粒度到按钮级别，权限分别菜单权限、数据权限
* 移动端
  - 导航栏默认有首页、我的两个导航
  - 底部导航栏动态通过后端Vue界面配置，根据不同的角色动态添加
  - 首页的菜单可以通过后端Vue界面配置，根据不同的角色动态添加

### 关键实现细节
* Vue前端、移动端
  - 数据字典相关的动态根据数据字典进行显示
* 移动端
  - 导航栏默认有首页、我的两个导航，其他中间位置的导航可以动态设置成中间凸起
